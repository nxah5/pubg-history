# PUBG OBS Tracker

PUBG API로 최근 매치의 내 팀 정보만 가져오고, 선택한 매치부터 평균 지표를 OBS 브라우저 소스로 출력하는 로컬 도구입니다.

## 실행

1. Node 18 이상으로 서버를 실행합니다.
2. 관리자 화면의 `API 키` 칸에 PUBG API 키를 저장합니다.
3. `샘플 데이터` 체크를 끄면 실 API로 검색합니다.

```bash
node server.js
```

npm이 있으면 아래 명령도 사용할 수 있습니다.

```bash
npm start
```

`.env`에 키를 고정해둘 수도 있습니다. `.env.example`을 `.env`로 복사한 뒤 `PUBG_API_KEY`에 키를 넣으면 됩니다.

샘플 데이터로 화면을 먼저 확인하려면:

```bash
PUBG_USE_MOCK=1 node server.js
```

## URL

- 관리자 화면: `http://127.0.0.1:3000/`
- 일반전 OBS: `http://127.0.0.1:3000/overlay/normal`
- 경쟁전 OBS: `http://127.0.0.1:3000/overlay/ranked`
- 사용자지정 OBS: `http://127.0.0.1:3000/overlay/custom`

OBS 브라우저 소스에서는 배경 투명 옵션을 켜고, 필요한 출력 URL을 넣으면 됩니다.

## 동작 방식

- 서버/shard와 닉네임으로 최근 매치를 검색합니다.
- `샘플 데이터` 모드가 켜져 있으면 API 키와 관계없이 데모 데이터가 나옵니다.
- 매치 상세에서는 상대팀 없이 검색 플레이어가 속한 팀 번호, 팀원, 개인 스탯을 표시합니다.
- 출력 플레이어는 최대 12명까지 등록됩니다.
- 출력명은 관리자 화면의 출력 플레이어 카드에서 수정합니다.
- `기록 시작`을 누른 매치부터 평균값을 계산합니다.
- `새 매치 갱신`은 기준 플레이어의 최근 매치를 다시 확인해, 시작 매치 이후 같은 출력 형태의 매치를 추가합니다.
- 활성 기록과 다른 OBS 링크는 빈 화면으로 표시됩니다.

## 평균 계산

- 평균킬뎃: 매치별 K/D 값을 더한 뒤 기록된 판수로 나눕니다.
- 평균딜량: 매치별 딜량을 더한 뒤 기록된 판수로 나눕니다.
- 평균이동거리: 매치별 이동거리를 더한 뒤 기록된 판수로 나눕니다.
- 이전판 생존 시간: 가장 최근에 반영된 매치의 생존 시간입니다.

## 사용자지정 점수

사용자지정 OBS에는 이스포츠 룰셋 점수를 추가로 표시합니다.

- 누적 킬: 기록된 사용자지정 매치에서 해당 플레이어 팀이 기록한 총 킬입니다.
- 팀 생존점수: 팀 생존순위 점수를 누적합니다.
- 팀 킬점수: 팀 킬 1개당 1점으로 누적합니다.
- 토탈점수: 팀 생존점수와 팀 킬점수를 더합니다.

생존점수는 1등 10점, 2등 6점, 3등 5점, 4등 4점, 5등 3점, 6등 2점, 7~8등 1점, 9등 이하는 0점입니다.

## API 참고

PUBG API는 JSON:API 형식으로 응답하며, 플레이어 검색은 `filter[playerNames]`, 매치 상세는 `/matches/{matchId}` 구조를 사용합니다. 매치 데이터 보존 기간은 14일이고, 기본 개발용 rate limit은 10 requests/minute입니다.
